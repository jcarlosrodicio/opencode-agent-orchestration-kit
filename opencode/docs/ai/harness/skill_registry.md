# Skill Registry

> Generated by `node scripts/update-skill-registry.mjs --check`.
> Do not edit manually. Re-run the generator after adding/removing skills.

## Purpose

The `lead` agent consults this registry before delegating non-trivial work. It selects 0-3 relevant skills per handoff and includes them in the `Skill Resolution` block. This reduces noise and saves tokens compared to injecting all skills.

## Public package boundary

This public registry includes project skills only. User-installed skills from
`~/.agents/skills` are intentionally omitted from committed public artifacts.

## Fallback

If this file is missing, `lead` continues using the global `<available_skills>` block as before. No agent should fail solely because this registry is absent.
