# v1.0.27 - Safe lifecycle and canonical release identity

This release candidate completes the unpublished changes since v1.0.26. It
does not claim that a tag or hosted release already exists.

## Highlights

- Complete CI coverage for contract checks, script tests, dependency audit,
  typechecking, and the installation smoke lifecycle.
- A manifest-owned installation lifecycle with dry-run planning, conflict
  detection, preserved user files, transactional rollback, and crash recovery.
- One canonical stable version in the root package, exposed consistently by
  every lifecycle entrypoint and recorded in all manifest snapshots.
- Version-aware doctor and upgrade rules that block downgrade and ambiguous
  same-version payload changes before mutation.
- A documented migration path for legacy installations through inspection,
  reviewed collisions, and explicit force adoption when intended.

## Installation or upgrade

For a new installation, preview the complete ownership plan before applying it:

```bash
./install.sh --dry-run
./install.sh
```

For an installation that already has a valid ownership manifest, preview and
apply the version-aware upgrade:

```bash
./upgrade.sh --dry-run
./upgrade.sh
./doctor.sh
```

These commands use the local checkout. This note does not claim that v1.0.27 is
already available as a remote tag or hosted release.

## Migration

Legacy installations without an ownership manifest use the normal initial
install path: back up the target, inspect `install --dry-run`, review every
collision, and use `install --force` only for regular-file replacements that
should intentionally become kit-owned.

There is no automatic migration for an unpublished experimental
`schema_version: 1` manifest that lacks `kit_version`. Back up the complete
target, review the exact `.oak` recovery state, remove only that experimental
state explicitly, and reinstall. The manager never invents a version or adopts
ownership from an invalid manifest.

## Validation performed

Release readiness was validated with the version contract, 295 differentiated
tests, exact S001-S136 and V001-V027 scenario IDs, typechecking, the configured
moderate-threshold dependency audit, installation smoke, shell syntax checks,
tag validation, diff checks, and the public leak scan. The audit reported three
low-severity transitive findings below the configured failure threshold.

## Public safety boundary

The public kit excludes local configuration, credentials, provider and MCP
wiring, runtime state, raw transcripts, private evidence, and machine-local
paths. Publication, tag creation, and hosted release creation remain manual
steps outside these repository changes.
