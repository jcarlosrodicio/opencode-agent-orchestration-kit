# v1.0.28 - Safe lifecycle and canonical release identity

This release adds an evidence-backed compatibility contract for the public
OpenCode orchestration kit while preserving the existing safe installation and
upgrade lifecycle.

## Highlights

- A canonical `compatibility.json` contract for supported Node.js lines,
  OpenCode boundaries, and exact plugin SDK and OpenTUI pins.
- Blocking CI coverage for Ubuntu on Node.js 22 and 24, macOS on Node.js 24,
  and isolated OpenCode core smokes for versions 1.14.41 and 1.18.4.
- A separate scheduled and manually dispatchable Node.js 26 / OpenCode latest
  canary whose failures remain visible without weakening release checks.
- A public compatibility matrix that distinguishes tested, supported,
  experimental, and unsupported surfaces, including WSL2, native Windows, and
  optional integrations.
- Credential-free compatibility smokes that copy only the packaged harness,
  isolate HOME, XDG state, npm cache, and working directory, and reject private
  path leakage.

## Installation or upgrade

For a new installation, preview the complete ownership plan before applying it:

```bash
./install.sh --dry-run
./install.sh
```

For an existing manifest-owned installation, preview and apply the upgrade:

```bash
./upgrade.sh --dry-run
./upgrade.sh
./doctor.sh
```

The supported runtime contract is Node.js `^22.9.0 || ^24.0.0` with OpenCode
`>=1.14.41 <2.0.0`. See `docs/compatibility.md` for the complete evidence and
status matrix.

## Migration

No compatibility-data migration is required. Existing ownership manifests and
preserved user files keep their current lifecycle semantics. Operators moving
from an older kit version should follow the normal version-aware upgrade path
and review `upgrade --dry-run` before mutation.

Native Windows remains unsupported because the lifecycle wrappers require
Bash. WSL2 is experimental and should be treated as a Linux-hosted path rather
than a native Windows guarantee.

## Validation performed

Release readiness was validated with 437 bundled tests, the version and
compatibility contracts, Node.js and OpenCode boundary smokes, TypeScript
typechecking, the configured moderate-threshold dependency audit, installation
smoke, diff checks, and the public leak scan. The audit reports three known
low-severity transitive findings below the configured failure threshold.

The blocking GitHub workflow exercises Ubuntu on Node.js 22 and 24, macOS on
Node.js 24, and OpenCode 1.14.41 and 1.18.4. The moving latest canary remains a
separate non-blocking observation.

## Public safety boundary

The public package excludes private OpenCode configuration, local provider and
MCP wiring, credentials, authentication files, raw transcripts, private
evidence, user-specific absolute paths, and machine-local state. In particular,
the local SearXNG research integration is not part of this release.
