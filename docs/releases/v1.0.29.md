# v1.0.29 - Safe lifecycle and canonical release identity

This release adds a reviewed reproducibility and supply-chain contract for the
public OpenCode orchestration kit while preserving the existing installation,
upgrade, rollback, and compatibility guarantees.

## Highlights

- A canonical `supply-chain.json` inventory ties every active external plugin,
  GitHub Action, Docker image, npm override, and tool version to an immutable
  reviewed identifier.
- Automated npm installs are frozen with `npm ci --ignore-scripts`; the complete
  lockfile is checked for canonical registry URLs and SHA-512 integrity, and the
  release gate now requires both vulnerability and registry-signature audits.
- The publishable npm surface is explicit. A new package smoke validates
  checksums, archive paths and entry types, forbidden local state, package
  installation, lifecycle behavior, and SDK import before release.
- OpenCode compatibility evidence is separated into core mode without optional
  integrations and default mode using the exact packed starter configuration.
- Blocking CI uses immutable Action commits and runs signature and package
  verification on the canonical Ubuntu/Node.js 24 lane without publishing or
  granting write permissions.
- The public supply-chain runbook documents reviewed pin updates and requires
  publishing the exact tarball that passed checksum and package smoke checks.

## Installation or upgrade

For a new installation, preview the complete ownership plan before applying it:

```bash
./install.sh --dry-run
./install.sh
```

For an existing manifest-owned installation, preview and apply the upgrade:

```bash
./upgrade.sh --dry-run
./upgrade.sh
./doctor.sh
```

The supported runtime contract remains Node.js `^22.9.0 || ^24.0.0` with
OpenCode `>=1.14.41 <2.0.0`. See `docs/compatibility.md` for the complete
evidence and status matrix.

## Migration

No ownership-manifest or configuration migration is required. Existing
manifest-owned installations retain their current lifecycle semantics and can
follow the normal version-aware upgrade path.

Operators preserving a locally modified managed file should continue to review
the dry-run merge state before acknowledging or applying an upgrade.

## Validation performed

Release readiness was validated with 612 bundled tests, the version,
compatibility, and supply-chain contracts, TypeScript typechecking, zero known
dependency vulnerabilities at the configured low threshold, 128 verified
registry signatures, 29 verified attestations, installation smoke, and the
exact npm package smoke.

Core compatibility smokes pass against OpenCode 1.14.41 and 1.18.4. The default
packed configuration smoke passes against OpenCode 1.18.4. The moving
OpenCode-latest canary remains separate and non-blocking.

The Docker adapter uses an immutable Node image digest, but Debian APT metadata
and BuildKit frontend behavior remain documented external reproducibility
limits.

## Public safety boundary

The public release excludes private OpenCode configuration, local provider and
MCP wiring, credentials, authentication files, raw transcripts, private
evidence, user-specific absolute paths, machine-local state, and internal Slice
1.5 planning artifacts. In particular, the local SearXNG research integration
is not part of this release.
